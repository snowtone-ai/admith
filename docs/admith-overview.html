Admith

Admithは、企業間の食品廃棄物を「捨てるコスト」から「再利用できる取引」に変えるためのMVPです。

何を解決するか

食品メーカーは、廃棄予定の食品副産物や余剰品を登録できます。
飼料・肥料・アップサイクル事業者は、条件に合う資源を取引候補として扱えます。
交渉、承認、決済記録、監査ログを一つの流れで残せます。

重要な設計思想

Admithでは、AIに契約判断を任せません。価格、制約、合意条件は決定論的なRule Engineで確定します。AIは将来的に候補整理や説明補助に使えますが、契約を確定する権限は持ちません。

署名・決済・契約確定の前には、必ず人間の最終承認を挟みます。これは企業利用、法規制、監査可能性を前提にした安全装置です。

基本の業務フロー

  1. 企業オペレーターが食品廃棄物リソースを登録する。
  2. システムがリソースをロックし、二重取引を防ぐ。
  3. Rule Engineが価格・法令・利用用途の条件を検証する。
  4. 合意案ができたら、人間の承認待ちにする。
  5. 承認後にSettlementへ進み、監査ログとフィードバック記録を残す。

画面でできること

Dashboard
交渉数、承認待ち、Delta金額を確認します。
Resources
食品廃棄物を登録し、交渉を開始します。
Negotiations
交渉状態、合意条件、承認画面への導線を確認します。
Audit
取引の時系列ログを確認します。

現在の位置づけ

このMVPは、企業デモと設計検証のためのPhase 0です。本番運用前には、組織ユーザー認証、KYB運用、DB永続化、権限管理、秘密情報管理、外部契約・マニフェスト連携の本番実装が必要です。

ローカル起動

.env に API_KEY を設定し、docker compose up -d --build を実行します。画面は http://localhost:3000 です。
