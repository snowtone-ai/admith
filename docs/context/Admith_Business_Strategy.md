## 1. 本質命題

> **「取引コスト（時間・情報処理・交渉帯域）が価値を上回るために市場が成立しなかったリソースを、AIエージェントの高速・並列・自律交渉により経済圏に引き込み、新市場を創出する」**

- 優位性は「既存市場と競合しない新粒度・新速度・新組み合わせ」
- 人間の認知速度・交渉帯域がボトルネックで死蔵されていたリソースが主戦場
- Adam Smithの「見えざる手」をデジタルリソースに適用するプロダクト

## 2. プロダクト体系

### 2.1 Admith（マーケットプレイス）

- **役割:** Admith自身が市場を運営。流動性・信頼・決済を一元提供
- **対象:** Cat.5（新市場創出・現技術で実装可能な20リソース）
- **収益モデル:** 創造価値の20%（成果報酬）
- **アナロジー:** Amazon型の市場支配

### 2.2 Admith Accord（プロトコル/インフラ）

- **役割:** 既存プラットフォーム・業界プレイヤーにエージェント交渉プロトコルをAPI提供
- **対象:** Cat.4（既存市場あり・参入余地ありの23リソース）
- **収益モデル:** API従量課金 + Trust-as-a-Service（信頼スコア利用料）
- **アナロジー:** Stripe / TCP-IP型のデファクトスタンダード化
- **展開時期:** Phase 0で実績を作ってから外部提供

### 2.3 フライホイール（相互強化ループ）

```
Accord導入企業増加
→ 共通プロトコルの普及
→ Admithマーケットプレイスの流動性向上
→ 取引データ蓄積
→ 価格指標データの精度向上
→ Accord利用者へのデータフィードバック
→ Accord価値向上 → さらに導入増加
```

## 3. 収益モデル（4本柱）

1. **最適化ヤイールド** — 取引額でなく「価値の増分（Delta）」に課金（負→正変換の差額の20%）
2. **Trust-as-a-Service** — エスクロー＋紛争解決の保証料（保険モデル）
3. **Sandbox従量課金** — エージェント戦略のA/Bテスト環境（AWS型）
4. **価格指標データライセンス** — 新経済指標をBloomberg的に販売

## 6. ユニットエコノミクス（Phase 0）

```
平均ロットサイズ:   5.0トン/取引
廃棄コスト:        ¥20,000/トン（= ¥100,000/取引の負の価値）
変換後価値:        ¥8,000/トン（= ¥40,000/取引の正の価値）
Delta:             ¥28,000/トン（= ¥140,000/取引の創造価値）

Admith収益（20%）:   ¥28,000/取引
AIコスト:            ¥11/取引
月間固定費:          ¥12,000,000（エンジニア5名・セールス2名・管理費）
BEP:                429件/月

市場TAM（食品廃棄物単体）: ¥33.6億/年
横展開後TAM:              兆円規模
```

**注意:** 上記は粗利ベース。営業/CS/法務/現地確認コストを含めると実際のBEPは上振れする。

## 7. 競合マップ

| 競合 | 脅威度 | Admithの防衛 |
|---|---|---|
| Too Good To Go | 高（2026年日本参入済、2027年B2B予測） | B2B多者間同時交渉に特化。TGTGのB2Cモデルでは真似できない複雑さ |
| ヤマト・佐川 | 中（AI導入済、生産性20%向上） | オープンプロトコルで大手物流をエコシステム参加者として取り込む |
| Big Tech | 高（2026-2027年に汎用エージェントプロトコル） | ドメイン固有パラメータ（おからの含水率等）の理解で差別化 |
| 内部は独自最適化、外部接続はA2A/MCP adapterで吸収 | - | ChatGPT監査で確定した戦略方針 |

## 8. 致命的リスクと対策

| リスク | 対策 |
|---|---|
| 規制による即死 | Phase 0は規制リスク低領域（フードロス）から開始 |
| Big Techの後追い | プロトコル層をオープン標準化しエコシステム先行構築 |
| コールドスタート問題 | 初期は招待制10社。シミュレーテッド・エージェント投入 |
| エージェント暴走 | Human Final Approval + サーキットブレーカー |
| Sybil価格操作 | KYB必須・招待制・wash trading検知 |
| プライバシー訴訟 | 差分プライバシー + ユーザー細粒度開示設定 |
| TGTG B2B参入 | B2B多者間同時交渉（飼料・物流・CC 3者同時）に特化 |
| 推論コスト急落でモート消失 | プロトコル設計・信頼スコア・交渉アルゴリズムで差別化 |
| 下請法→取適法改正（2026年1月） | 自動交渉でも協議ログ・見積根拠・変更理由を保存 |

## 10. Phase 2-3 将来構想

### 意図の先物（Phase 2）
- ユーザーの確率的意図（「引越し確率72%」等）をエージェントがブロードキャスト
- 不動産・家具エージェントが意図情報に入札（成約前の段階）
- 月額¥2,000〜3,000の収益実証が日本ユーザー採用閾値
- 広告の「行動後ターゲティング」→「意図前取引」へのパラダイム転換
- 3層オプトイン: Identifier（常に匿名/ZKP）、Broad Intent（差分プライバシー）、Constraint（暗号化）

### コンテキスト・デルタ（Phase 2）
- 単体では無価値だが結合で価値爆発する情報断片の取引
- 大学TLO（技術移転オフィス）が有望初期顧客
- Combination Value Engine（ベクトル埋め込み + Knowledge Graph）が核心技術

### 制度バッファ（Phase 3）
- 医療ベッド・電力予備容量等の安全マージン余剰を流動化
- 規制当局との協働が必須。社会インフラの動的最適化

### 抽象的リソース（Phase 3+）
- 認知エントロピー: 組織の思考均質性を崩す異質知見の定量調達
- 関係性トポロジー: 個人情報なしでネットワーク構造だけを貸し出す
- 不行使オプション価値: 「参入しない」ことが他者に生む価値の取引
- 規制解釈帯域: 法規制の解釈の幅そのものを取引
- 時間的アービトラージ権: 異時点間の価値差をリアルタイム裁定
