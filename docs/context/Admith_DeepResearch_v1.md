## Research 1：B2B食品廃棄物市場（Felo Web）

### 市場規模・成長性

- 食品ロス削減テクノロジー市場（AI需要予測・フードシェアリング・アップサイクル含む）は2030年に**世界で10兆円規模**に達すると予測
- 日本の食品リサイクル法対象となる「多量発生事業者」（年間100トン以上排出）は毎年6月末に定期報告義務あり。対象17項目を報告

### 日本の既存B2Bプラットフォーム動向

| サービス | 概要 | 備考 |
|---|---|---|
| **フーディール (Foodeal)** | 2024年4月から全国展開。フードバンクと食品提供事業者をマッチング。都道府県が主体となって導入 | 防災用備蓄食品の有効活用にも対応 |
| **BtoBプラットフォーム 商談** | 販路拡大向け商談プラットフォーム内にフードバンクコーナー設置 | 食品メーカー間の余剰在庫取引 |
| **WasteTrade** | デジタル廃棄物取引プラットフォーム（グローバル）。中間コスト削減・取引効率化が目的 | Admithが目指すモデルの先行事例 |
| **Full Harvest（米国）** | 規格外・余剰農産物に特化したB2Bマーケットプレイス | |
| **Cirplus（欧州）** | リサイクルプラスチックに特化したB2Bプラットフォーム | |

### 廃棄物取引のコスト構造

- 排出事業者〜処理業者間にブローカー・商社が介在する場合、処理費用に仲介手数料が上乗せ。契約プロセスが複雑化
- **物流コストの決定要因：** 運搬回数（トラック台数）× 輸送距離 + 在庫管理・包装・人件費
- 2024年問題（トラックドライバー時間外労働規制強化）により物流コスト大幅増加リスク。廃棄物処理コスト上昇に直結
- 実例：廃棄物処理費用を管理体制見直しにより**年間¥3,000万→¥1,500万に半減**した企業事例あり

### 食品リサイクル法の制約（Phase 0 設計に直結）

| 制約 | 内容 |
|---|---|
| **再生利用の義務** | 多量発生事業者（年間100トン以上）は飼料化・肥料化・メタン化等による再生利用目標が課せられる |
| **優先順位** | 飼料化 → 肥料化（この順序が法定されておりエージェントの目的関数に組み込む必要あり） |
| **報告と罰則** | 目標未達の場合、勧告→公表→命令。命令違反で罰則 |
| **不適正転売の防止** | 肥後物産事件を受け、転売に厳しい監視。トレーサビリティ確保が必須 |

## Research 2：ZKP（ゼロ知識証明）プライバシー技術（Felo 論文）

### 主要発見

- ZKPはブロックチェーン・スマートコントラクトと統合し、ヘルスケア分野でプライバシー強化認証・アクセス管理の実装例が存在（2025年論文）
- IoT活用ヘルスケア環境でのZKP＋スマートコントラクト実装：データ完全性**92%向上**、プライバシー保護**87%向上**、不正アクセス**30〜35%減少**
- 連合学習（FL）との組み合わせで、データポイズニング攻撃への堅牢性が既存FLと比較して**15〜25%向上**

### ZKP vs 連合学習の役割分担

| 技術 | 主な役割 | 特徴 |
|---|---|---|
| **ZKP** | 認証・アクセス管理 | 「データの性質が正しい」ことを証明。データ内容は非開示 |
| **連合学習** | 分散型協調AIトレーニング | 生データを集めずモデル訓練。プライバシーを維持しながら学習 |
| **SMPC** | 共同計算 | 互いのデータを隠したまま「結合時の相関値」を計算 |

### 実装コストに関する注意

**論文ではZKPの具体的なベンチマーク（証明生成時間・コスト）やUX調査の記述なし。** 実装コストの詳細数値は別途調査が必要。現時点での判断：Phase 0・Phase 1ではZKPは不要。Phase 2（意図データ取引）で必要になった時点で実装コストを再評価する。

## Research 3：AIエージェントの法的有効性（Felo Web、日本・EU）

### 日本法における契約能力

- **現行法：** AIエージェントに法的人格（権利能力）なし。法律上は「ツール」
- **電子署名法の解釈：** 「利用者の意思のみに基づいて機械的に暗号化が行われる」事業者署名型は、一定条件下で電子署名法上の電子署名に該当するとの政府見解あり → エージェントの行動が事前設定のルール範囲内であれば「本人の意思に基づく署名」として有効と解釈可能
- **リスク：** エージェントが予測不能な判断を下した場合、行為が利用者の意思の範囲内か否かの法的議論が継続中
- **表見代理リスク：** エージェントが権限を逸脱して契約を結んだ場合でも、相手方が正当な権限があると信じる理由があれば企業が責任を負う可能性

### EU AI Act（2026年適用開始）の要件

金融サービス・契約管理に用いられるエージェントが「高リスクAIシステム」に分類される可能性がある場合:
- **透明性の確保：** AIが契約を締結していることを取引相手に明示する義務
- **人間の監督：** AIシステムの運用を人間が適切に監督できる仕組みの導入
- **リスク管理：** AIシステムのライフサイクル全体を通じたリスク管理体制の構築

**重要：** AIエージェントによる自律的な契約の有効性に関する明確な規定はまだなく、法的な不確実性が残っている。

### 紛争解決メカニズム（現行・技術的対応）

| 手法 | 内容 |
|---|---|
| ログと監査証跡 | エージェントの意思決定プロセス・通信履歴を詳細記録 |
| 契約による事前合意 | プラットフォーム利用規約に紛争解決ルール（仲裁条項・準拠法）を明記 |
| 技術的監視と制御 | エージェントの行動を監視し、逸脱した行動を検知した際に人間が介入 |

**将来展望：** AIエージェントに限定的な法人格を与え保険加入を義務付ける枠組みが議論されているが、実現には多くの課題が残る。

### Admithへの設計含意

- Phase 0では「エージェントは企業の代理人」として設計。権限範囲をシステム上で厳密に設定できる機能が必須
- 利用規約でエージェント間メッセージ一致時点を契約成立と明記
- マルチエージェントシステムでは通信の盗聴・改ざん・なりすましリスクに対する暗号化が不可欠

## Research 5：データ収益化ユーザー心理（Felo SNS）

### 主要発見

**データ販売への心理的障壁：**
- GmailなどのAIによるメール内容解析に対し、プライバシー侵害の懸念がSNS上で多数確認
- ユーザーが直接データ販売から利益を得る仕組みは一般的でなく、インセンティブが不足
- OpenAIはChatGPTに広告テストを行う際「ユーザーデータを広告主に販売しない」と明言し、ユーザーの信頼維持を優先

**AIエージェントによるデータ収益化への反応：**
- AIエージェントがユーザーに代わってデータ取引・交渉を行うコンセプトへの期待あり
- 同時に「AIエージェントコミュニティを装った偽の投稿」「高額エージェントサービスが期待外れ」等の懐疑的見方も存在（不信感の根拠）
- 「AIをどのように実務・学習に役立てるか」への具体的な関心が高まっている

**日本市場特有のプライバシー感度：**
- マイナンバーカードの情報を「全て渡すのではなく、年齢確認に必要な情報のみ」に限定する技術が採用されており、**最小限の情報開示**への需要が日本で高い
- 企業がAIを利用する際の倫理・責任を求める声がある
- SNS上でのネガティブコメントのストレスを避けたいという意識が利用プラットフォームの選択に影響

**SNS調査の限界：**  
データ収益化サービス（Datacoup, Wibson等）への実際のユーザー反応、「エージェントが代わりに稼ぐ」フレーミングへの具体的な反応データは今回の検索では不足。より定量的なデータ（調査レポート・論文）の追加調査を推奨。

### Admithへの設計含意

- 「データを売る」という表現は心理的障壁が高い。「**自分のAIエージェントが代わりに稼ぐ**」というフレーミングに統一する
- 日本ユーザーの「最小限開示」志向に合わせ、3層のオプトイン粒度（Identifier/Broad Intent/Constraint）を実装（詳細はHandoff v2 §8参照）
- 初期段階では最も抵抗の少ない属性（広域な地域・年齢層）からオプトイン開始
